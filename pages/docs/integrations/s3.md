# S3-Compatible Object Storage Integration

This guide explains how to use `forze_s3` with Amazon S3, MinIO, and other S3-compatible services.

## Prerequisites

- `forze[s3]` installed
- bucket already exists
- credentials have read/write access

## What Forze handles vs does not handle

Forze handles:

- resolving `StoragePort` by bucket (`ctx.storage("bucket")`)
- object upload/download/delete/list operations
- metadata and content-type handling

Forze does not manage:

- bucket creation workflows in cloud/IaC
- IAM policies, ACLs, bucket policies
- lifecycle rules (retention/archive), CORS, encryption settings

## Runtime wiring

    :::python
    from forze.application.execution import DepsPlan, ExecutionRuntime, LifecyclePlan
    from forze_s3 import S3DepsModule, S3Client, s3_lifecycle_step

    s3_client = S3Client()
    s3_module = S3DepsModule(client=s3_client)

    runtime = ExecutionRuntime(
        deps=DepsPlan.from_modules(s3_module),
        lifecycle=LifecyclePlan.from_steps(
            s3_lifecycle_step(
                endpoint="https://s3.amazonaws.com",
                access_key_id="your-access-key",
                secret_access_key="your-secret-key",
            )
        ),
    )

For MinIO:

    :::python
    s3_lifecycle_step(
        endpoint="http://localhost:9000",
        access_key_id="minioadmin",
        secret_access_key="minioadmin",
    )

Optional client tuning can be passed via `S3Config` (botocore config-compatible dict).

## Resolve and use storage port

    :::python
    storage = ctx.storage("app-assets")

    stored = await storage.upload(
        filename="invoice.pdf",
        data=pdf_bytes,
        description="Invoice #42",
        prefix="invoices/2026/03",
    )

    downloaded = await storage.download(stored.key)
    await storage.delete(stored.key)

    objects, total = await storage.list(limit=20, offset=0, prefix="invoices/2026")

## Storage operation reference

| Method | Returns |
|--------|---------|
| `upload(...)` | `StoredObject` (`key`, `filename`, `content_type`, `size`, `created_at`) |
| `download(key)` | `DownloadedObject` (`data`, `content_type`, `filename`) |
| `delete(key)` | `None` |
| `list(limit, offset, prefix=...)` | `(list[StoredObject], total_count)` |

## Multi-tenant behavior

If a tenant context dependency is registered, S3 storage adapter automatically incorporates tenant context when building object paths.
