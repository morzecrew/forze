# S3-Compatible Object Storage Integration

This guide explains how to set up S3-compatible object storage for Forze. It covers connection configuration, the storage port, and object operations. Supports Amazon S3, MinIO, Yandex Cloud Object Storage, and other S3-compatible services.

## Prerequisites

- S3-compatible storage service
- `forze[s3]` extra installed (includes `aioboto3`, `python-magic` for content-type detection)

## Out of Scope

**The framework does not manage infrastructure or bucket configuration.** You must handle the following outside Forze:

- **Bucket creation** — Create buckets via your cloud provider's console, CLI, or IaC (Terraform, CloudFormation, etc.) before using the storage port.
- **ACLs and access policies** — Configure bucket ACLs, bucket policies, and IAM permissions separately.
- **Lifecycle rules** — Object expiration, transitions, and archival are not configured by the framework.
- **CORS, versioning, encryption** — Bucket-level settings are your responsibility.

The storage adapter assumes buckets already exist and that the configured credentials have read/write access to them.

## Connection

Use :class:`S3DepsModule` to register the client and storage port, and :func:`s3_lifecycle_step` for startup/shutdown:

```python
from forze.application.execution import Deps, LifecyclePlan
from forze_s3 import S3Client, S3DepsModule, s3_lifecycle_step

client = S3Client()
deps_module = S3DepsModule(client=client)

# Build deps and lifecycle
deps = deps_module()
lifecycle = LifecyclePlan.from_steps(
    s3_lifecycle_step(
        endpoint="https://s3.amazonaws.com",
        access_key_id="your-access-key",
        secret_access_key="your-secret-key",
    )
)
```

For MinIO or other S3-compatible endpoints:

```python
s3_lifecycle_step(
    endpoint="http://localhost:9000",
    access_key_id="minioadmin",
    secret_access_key="minioadmin",
)
```

See :class:`forze_s3.kernel.platform.client.S3Config` for optional botocore config (retries, timeouts).

## Port Registered

:class:`S3DepsModule` registers :data:`StorageDepKey` with a factory that builds :class:`StoragePort` instances per bucket.

## Storage Port

Resolve a storage port via :meth:`ExecutionContext.storage`:

```python
storage = ctx.storage("my-bucket")
```

### Operations

| Method | Description |
|--------|-------------|
| `upload(filename, data, description=..., prefix=...)` | Upload bytes; returns :class:`StoredObject` with key, filename, content-type, size, created_at |
| `download(key)` | Download object by key; returns :class:`DownloadedObject` with data, content-type, filename |
| `delete(key)` | Delete object by key |
| `list(limit, offset, prefix=...)` | List objects with pagination and optional prefix filter |

### Key Generation

Uploaded objects receive a UUIDv7-based key. Use `prefix` to organize objects (e.g. `prefix="uploads/2024"`). Keys are built with :class:`forze.utils.codecs.PathCodec`.

### Metadata

The adapter stores `filename`, `created_at`, `size`, and optional `description` in object metadata. Content type is inferred via `python-magic` or `mimetypes`; fallback is `application/octet-stream`.

## Checklist

- [ ] Bucket created and accessible with your credentials
- [ ] ACLs and IAM policies configured for read/write
- [ ] `forze[s3]` extra installed
- [ ] S3 client initialized via `s3_lifecycle_step` before operations
- [ ] `S3DepsModule` merged into your deps
